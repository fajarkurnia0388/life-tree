## 2. Fungsi Dokumen

Dokumen ini menetapkan kerangka metodologis untuk seluruh arsip riset cultivation. Ruang lingkupnya mencakup:

1. definisi kerja tentang cultivation;
2. batas objek kajian;
3. jenis sumber yang dipakai;
4. hubungan antara tradisi, genre, dan toolkit kreatif;
5. aturan terminologi dan transliterasi;
6. prinsip penggunaan karya benchmark.

Dokumen ini tidak memuat isi topik cultivation spesifik secara penuh. Seluruh pembahasan substantif tetap berada pada file domain lain.

---

## 3. Definisi Kerja

Dalam arsip ini, istilah **cultivation** dipakai sebagai kategori payung untuk karya, sistem, dan worldbuilding yang menampilkan satu atau lebih ciri berikut:

1. **progresi eksistensial bertingkat** (realm, stage, breakthrough, ascension);
2. **pengolahan energi/essence/spirit** secara sadar dan sistemik;
3. **hubungan eksplisit dengan hukum kosmik, Dao, karma, atau struktur transendensi**;
4. **konsepsi tubuh sebagai wadah transformasi metafisik**;
5. **pengintegrasian institusi, ekonomi, dan konflik dalam tatanan kekuatan pasca-manusia**.

Definisi ini mencakup xianxia dan xiuzhen secara utama, serta mencakup varian hibrida selama inti progresi cultivation tetap dominan.

## 5. Objek yang Masuk dan Tidak Masuk

### 5.1 Objek yang masuk
- xianxia klasik;
- xiuzhen / xiuxian modern;
- xuanhuan yang masih memakai struktur cultivation eksplisit;
- cultivation comedy / meta cultivation;
- dark / demonic / horror cultivation;
- system cultivation yang mengadopsi struktur cultivation sebagai inti progresi;
- urban cultivation;
- interstellar / techno-cultivation;
- karya lintas media yang relevan: novel, manhua, donghua, drama, game, audio, dan adaptasi terkait.

### 5.2 Objek yang tidak otomatis masuk
- wuxia non-cultivation murni;
- fantasy Tiongkok tanpa struktur cultivation yang bermakna;
- karya dengan unsur spiritual umum tetapi tanpa sistem progresi cultivation;
- karya yang hanya memakai istilah “Dao”, “Qi”, atau “Immortal” secara dekoratif tanpa fungsi sistemik.

Keputusan ambigu dibahas per kasus dalam `META-004` dan `COMP-001`.

## 8. Aturan Korpus Benchmark

### 8.1 Fungsi korpus benchmark
Korpus benchmark dipakai untuk:
- memberi contoh konkret lintas file;
- menjaga konsistensi analisis komparatif;
- mencegah file dibangun hanya dari satu karya favorit;
- menjaga keseimbangan antara karya klasik, modern, arus utama, dan hibrida.

### 8.2 Prinsip pemilihan benchmark
Karya benchmark dipilih berdasarkan kombinasi faktor berikut:
- pengaruh historis;
- representativitas subgenre;
- kejelasan sistem cultivation;
- kekuatan worldbuilding;
- nilai komparatif;
- nilai market / adaptasi / fandom bila relevan.

### 8.3 Status korpus
Korpus benchmark bersifat **Living**, tetapi perubahan tidak dilakukan terlalu sering. Revisi korpus harus dijustifikasi dan dicatat.

## 10. Aturan Scope Writing

Setiap file wajib menyatakan tiga hal:

1. **pertanyaan inti file**;
2. **apa yang dibahas**;
3. **apa yang tidak dibahas**.

Tujuannya adalah mencegah:
- pengulangan definisi inti di terlalu banyak file;
- file satelit tumbuh menjadi file hub tanpa kendali;
- konflik owner concept.

Aturan implementasi rinci berada di `META-003`.

---

## 11. Aturan Cross-Reference

- File owner selalu dirujuk ketika konsep inti disentuh oleh file non-owner.
- Definisi inti tidak boleh ditulis ulang penuh tanpa kebutuhan analitis yang jelas.
- File comparative tidak mengambil alih fungsi file definisional.
- File market tidak menggantikan file foundation atau core.
- File toolkit tidak boleh diperlakukan sebagai bukti sejarah atau bukti tradisi.

## 13. Risiko Metodologis yang Harus Dijaga

| Risiko | Dampak | Mitigasi |
|---|---|---|
| Campur aduk tradisi dan trope | kesimpulan rancu | pemisahan lapisan analisis |

| Data pasar basi | file market kehilangan nilai | status Living/Snapshot |
