# Rencana Topik Riset Terpisah V2 — Immortal Cultivation / Xianxia / Xiuzhen

Dokumen ini adalah **versi revisi dan perluasan total** dari rencana riset sebelumnya.

Versi ini menggabungkan masukan terbaik dari empat evaluator, lalu **disesuaikan dengan preferensimu**: 
> **scope diperluas, bukan dipersempit** — tujuannya membuat arsip riset yang selengkap mungkin.

Namun, agar proyek sebesar ini tetap bisa dijalankan, struktur di bawah memakai prinsip:
- **scope luas**,
- **batas file jelas**,
- **fondasi metodologis kuat**,
- **prioritas bertahap**,
- **pemisahan topik stabil vs topik yang cepat usang**.

---

# FILOSOFI V2

Versi ini dibangun di atas 6 prinsip:

1. **Cultivation bukan satu topik, melainkan ekosistem.**
   Ia mencakup metafisika, tubuh, jiwa, hukum kosmik, politik, ekonomi, flora-fauna, narasi, pasar, fandom, dan IP.

2. **Tidak semua file punya fungsi yang sama.**
   Ada file fondasi, file hub, file satelit, file komparatif, file toolkit, dan file snapshot pasar.

3. **Riset lengkap tidak berarti semua file harus Level 3.**
   Kita boleh punya scope raksasa, tapi kedalaman harus bertingkat.

4. **Batas antar file harus eksplisit.**
   Karena proyek ini besar, overlap tidak dihindari dengan memperkecil scope, tetapi dengan memperjelas fungsi file.

5. **Ada perbedaan antara konsep klasik, konvensi genre, dan desain original.**
   Ketiganya akan dipisahkan secara sadar.

6. **Bagian pasar/tren/IP harus diperlakukan sebagai data hidup.**
   Berbeda dengan file metafisika atau worldbuilding yang lebih stabil.

---

# SISTEM KLASIFIKASI FILE

Agar tidak kacau, semua file dibagi menjadi 8 tipe:

- **[META]** metodologi, glosarium, bibliografi, korpus
- **[FOUNDATION]** fondasi genre dan filsafat
- **[CORE]** metafisika inti cultivation
- **[SYSTEM]** mekanik progresi, risiko, dan struktur kekuatan
- **[PATH]** jalur/jurusan/spesialisasi cultivation
- **[WORLD]** worldbuilding makro dan mikro
- **[ENTITY]** makhluk, flora, resource, artefak, relik
- **[NARRATIVE]** teknik bercerita cultivation fiction
- **[COMPARATIVE]** perbandingan antar karya/subgenre/era
- **[MARKET]** ranking, platform, IP, tren, fandom
- **[TOOLKIT]** template desain original

> Ya, totalnya memang lebih dari 8 label; beberapa file akan memakai label ganda.

---

# STATUS FILE

Setiap file nanti sebaiknya punya status:

- **Hub File** → file pusat, sering dirujuk
- **Satellite File** → file pendukung/topik khusus
- **Living File** → terus diperbarui
- **Snapshot File** → menangkap data pada periode tertentu
- **Stable File** → relatif final/stabil
