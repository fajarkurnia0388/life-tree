## 2. Tujuan Dokumen

Dokumen ini menetapkan dua hal utama:

1. **Minimum Viable Archive (MVA)** — subset minimum file yang harus tetap bernilai mandiri;
2. **Milestone proyek** — tahapan hasil yang menandai kematangan bertahap arsip.

Dokumen ini mencegah proyek besar kehilangan bentuk apabila penghentian sementara atau perlambatan terjadi.

---

## 3. Definisi Minimum Viable Archive (MVA)

MVA adalah himpunan file minimum yang, bila selesai terlebih dahulu, sudah cukup untuk menghasilkan:
- kerangka kerja arsip yang koheren;
- fondasi ontologis cultivation;
- struktur dasar worldbuilding;
- utilitas nyata bagi penulis, worldbuilder, atau analis genre.

MVA bukan “versi kecil” dari seluruh proyek; MVA adalah **inti fungsional proyek**.

## 5. Fungsi MVA

Jika MVA selesai, arsip sudah dapat dipakai untuk:

1. memahami cultivation sebagai sistem energi, ontologi, dan institusi;
2. membangun setting cultivation original yang tidak dangkal;
3. membedakan dasar genre dari trope populer semata;
4. menjadi landasan untuk ekspansi ke file satelit, comparative, dan market.

---

## 7. Aturan Penggunaan Milestone

- milestone dipakai untuk mengukur kematangan, bukan sekadar jumlah file selesai;
- sebuah milestone dianggap tercapai bila file utamanya tersedia minimal pada Level 2, kecuali dinyatakan lain;
- penyelesaian milestone harus tercermin di `META-001`.

---

## 8. Risiko Tanpa MVA

Tanpa MVA, proyek dapat mengalami masalah berikut:
- terlalu banyak file satelit tanpa pusat gravitasi;
- governance dan isi berkembang tidak seimbang;
- saat proyek berhenti sementara, tidak ada subset yang benar-benar berguna;
- arsip besar tetapi tidak operasional.

MVA berfungsi sebagai pencegah kondisi tersebut.
