🌱 Filosofi Anti-Guilt

Mengapa Streak Itu Beracun?

Aplikasi produktivitas konvensional mengandalkan loss aversion via sistem streak — memicu rasa bersalah saat Anda melewatkan 1 hari. Daoji menolak pendekatan ini.

❌ Aplikasi Konvensional

  • Streak rusak = kembali ke nol — hukuman visual
  • Guilt-driven: "Kamu merusak streak-mu!"
  • Overload tanpa peringatan — tambah habit terus
  • Satu domain saja: to-do list atau habit
  • Data tidak terenkripsi — bisa diakses server

✅ Daoji (Anti-Guilt)

  • Tidak ada streak — cumulation, bukan consecutive
  • Empathy-driven: "Pohonmu sedang beristirahat."
  • Canopy Load mencegah overload — soft enforcement
  • 6 domain kehidupan terintegrasi (Life Audit)
  • Lokal-Pertama (Privasi Penuh) — Data tersimpan lokal, tanpa server

⚙️ Fitur Inti

Arsitektur yang Dirancang untuk Empati

Setiap fitur dibangun berdasarkan prinsip: kegagalan adalah data, bukan hukuman.

🌿

Canopy Load System

Kapasitas harian terbatas (default 10 poin). Sistem mencegah overload dengan soft enforcement — peringatan ramah, bukan hard block. Automaticity Decay membebaskan ruang seiring waktu.

📋

Action of the Day

Algoritma rule-based memprioritaskan 1 habit berdasarkan domain terlemah. Priority Score = (domain_deficit × impact) / (friction + energy). Tanpa AI, tanpa black box.

📝

Friction Journaling

Saat habit terlewat, bukan hukuman — justru dipandu memahami hambatan. "Kurang Waktu?" → tawarkan versi mva_duration_min menit. "Kelelahan?" → tawarkan Recovery Mode.

🛡️

Keamanan Lokal & Enkripsi Roadmap

Database saat ini tersimpan 100% lokal di perangkat Anda (offline-first, tanpa akun server). Fitur enkripsi database (SQLCipher) sedang dikembangkan untuk rilis fase berikutnya.

🔍

Life Audit & Radar

6 domain kehidupan (Tubuh, Keuangan, Hubungan, Emosi, Karir, Rekreasi) dipetakan dalam Radar Keseimbangan. Domain terlemah mendapat prioritas otomatis.

🆘

Passive Crisis Protocol

Mood ≤ 2 selama 3 hari → modal empatik + Safety Card (119/SEJIWA hardcoded). Tidak ada AI/ML — false negative/false positive terlalu berisiko. Recovery Mode tersedia kapan saja.

🧠 Thinking Canvas

Thinking Canvas: Pilar Refleksi Berbasis Kertas

Daoji tidak menggantikan buku catatanmu. Saat pikiran kabur, Thinking Canvas menyarankan metode corat-coret di kertas asli — lalu membantumu menyimpan ringkasan dan memilih satu aksi kecil.

Mind Dump + Cluster + Refine

Ambil buku/kertas. Gunakan metode ini untuk mengurai pikiran penuh menjadi klaster, asumsi, dan satu langkah besok.

15 menit Harian ringan 1 aksi kecil
  • Ambil buku/kertas, lalu dump bebas 5–10 menit tanpa sensor.
  • Kelompokkan ide yang mirip.
  • Pilih 1 ide, catat asumsi, dan tulis langkah besok.
Paper-first: pikirkan di kertas, simpan ringkasannya di Daoji. Hasil akhirnya bisa menjadi Action of the Day, catatan Friction Journal, atau bahan Weekly Pulse.
